# Analytic Layouts(McKinsey 融合版)

本文补充 `layouts-guide.md`——后者是**叙事型版式**(封面、章节、结论陈述等),本文是**分析型版式**,用于展开 logic flow 和数据论证。两者合起来是完整的版式工具箱。

---

## 叙事型 vs 分析型

| 类型 | 用途 | 参考文件 |
|------|------|---------|
| **叙事型** | 封面、章节分隔、目录、结论陈述、KPI 聚焦 | `layouts-guide.md` |
| **分析型** | 展开论证、数据对比、逻辑推理 | 本文 |

一份典型的西井方案 PPT:**叙事型骨架 + 分析型内容**。章节之间用叙事型留白和陈述页,章节内部用分析型展开论证。

---

## 7 种分析型版式

### 1. 标题 + 单图表型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题(top-left)         │
│ ───────────                  │
│ ┌──────────────────────────┐ │
│ │   主图表(占 60–70%)      │ │
│ │                          │ │
│ └──────────────────────────┘ │
│ 数据来源:xxx   洞察:yyy     │
└──────────────────────────────┘
```

**适用**:单一数据故事、趋势展示
**图表类型**:折线图 / 柱状图 / 面积图

**Builder 映射**
- `pptx_builder.py` **没有**原生图表生成能力。
- 做法:**先用 matplotlib / plotly 生成图表 PNG**,保存到临时路径,然后用 `image(title, img_path, caption)` 或 `text_image(title, body, img_path)` 插入。
- 如果图表需要搭配一句洞察文字,用 `text_image`,把洞察写在 body。

**最佳实践**
- PNG 分辨率至少 1920×1080,保证放大不模糊
- 图表配色优先用 westwell 蓝(参考 `design-system.md` 的 `C_NAVY` / `C_TEAL`)而非 matplotlib 默认色
- 坐标轴、图例字号不小于 12pt(缩放后仍可读)
- 数据来源一行放在 body 最后(西井风格不做专门的 footer)

---

### 2. 标题 + 左右分栏型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题                    │
│ ───────────                  │
│ ┌──────────┐  ┌──────────┐  │
│ │ 左侧      │  │ 右侧      │  │
│ │ 图表/要点 │  │ 解释/案例 │  │
│ └──────────┘  └──────────┘  │
└──────────────────────────────┘
```

**适用**:一图一文、对比分析、图表+解读
**Builder 映射**
- 两边都是文字 → `two_col(title, left_head, left_body, right_head, right_body)`
- 左图右文 → `text_image(title, body, img_path)`(text_image 默认左图右文或左文右图,按 builder 定义)
- 左文右图 → 同上,调换

**最佳实践**
- 左右内容量尽量对称,避免一边空一边挤
- 对比类用 `two_col`,"图 + 解读"类用 `text_image`

---

### 3. 标题 + 2×2 矩阵型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题                    │
│ ───────────                  │
│         ↑ Y轴                │
│  ┌──────┬──────┐             │
│  │ 象限2 │ 象限1 │             │
│  ├──────┼──────┤             │
│  │ 象限3 │ 象限4 │             │
│  └──────┴──────┘  → X轴      │
└──────────────────────────────┘
```

**适用**:战略定位、竞争矩阵、方案选择
**西井典型用法**:竞品在"技术深度 × 场景专精度"矩阵里的分布、方案的"落地速度 × 覆盖广度"取舍

**Builder 映射**
- **没有原生 2×2 方法**
- 做法:用 matplotlib / 手绘工具生成矩阵 PNG,`image()` 或 `text_image()` 插入
- 或者用 `table(title, headers=["", "X 低", "X 高"], rows=[["Y 高", ..., ...], ["Y 低", ..., ...]])` 快速做出表格版,不如图美但快

**最佳实践**
- 象限标签要简短(≤ 6 字),象限内可放 2–3 个点(例如竞品 logo + 名称)
- 必须在 body 或 caption 里明确"我们选哪个象限、为什么"(BCG strategic choice 原则)

---

### 4. 标题 + 表格型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题                    │
│ ───────────                  │
│ ┌─────┬─────┬─────┬─────┐    │
│ │ 维度│ 方案A│ 方案B│ 方案C│    │
│ ├─────┼─────┼─────┼─────┤    │
│ │ ... │ ... │ ... │ ... │    │
│ └─────┴─────┴─────┴─────┘    │
└──────────────────────────────┘
```

**适用**:多维度对比、竞品对照、方案评分
**Builder 映射**:`table(title, headers, rows, bold_col=0)`

**最佳实践**
- 3–5 列、4–6 行是黄金区间,再多就难读
- 第一列作为维度名,`bold_col=0` 加粗
- 单元格内只放一个值或短语,不要放句子
- 如果要突出"我们推荐的列",可以在表头文本前加 ★ 或在 caption 里明说

---

### 5. 标题 + 瀑布图型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题                    │
│ ───────────                  │
│  ┌──┐                         │
│  │  │─┐ ┌─┐                   │
│  │  │ └─┤ ├─┐                 │
│  │起│   │增│ │终│              │
│  │点│   │量│ │点│              │
│  └──┘   └─┘ └─┘              │
└──────────────────────────────┘
```

**适用**:增长 / 变化归因拆解
**西井典型用法**:方案上线后"效率提升从 A 到 B,由哪些因素贡献"

**Builder 映射**
- **没有原生瀑布图方法**
- 做法:matplotlib 有 waterfall 插件,或用 plotly 的 `go.Waterfall`,生成 PNG 后 `image()` 插入

**最佳实践**
- 起点 / 终点用深色(navy),增量用 teal,减量用灰或暖色
- 每个柱子上标数值,不要让观众去读坐标轴

---

### 6. 标题 + 时间轴型

**结构**
```
┌──────────────────────────────┐
│ 论点式标题                    │
│ ───────────                  │
│  阶段1 ─── 阶段2 ─── 阶段3    │
│   │        │        │         │
│   下部:    下部:    下部:     │
│   内容     内容     内容      │
└──────────────────────────────┘
```

**适用**:历史演进、阶段划分、实施路线图
**西井典型用法**:四阶段孵化路径(PoC → 小试 → 推广 → 规模化)

**Builder 映射**
- **没有原生时间轴方法**
- 最接近的原生方案:`three_col(title, columns=[{head,body},...])` ——三阶段路线图
- 四阶段或更多:生成时间轴 PNG,`image()` 或 `text_image()` 插入
- 或者用 `table`,第一行是阶段名,第二行是目标,第三行是关键指标

**最佳实践**
- 每个阶段必须有可测量的成效指标,不能只写"完成 X"
- 阶段数 ≤ 4,再多信息密度就过载

---

### 7. 洞察总结页

**结构**
```
┌──────────────────────────────┐
│ 总结性标题                    │
│ ───────────                  │
│ ┌────┐ ┌────┐ ┌────┐          │
│ │ 01 │ │ 02 │ │ 03 │          │
│ │标题│ │标题│ │标题│          │
│ │说明│ │说明│ │说明│          │
│ └────┘ └────┘ └────┘          │
└──────────────────────────────┘
```

**适用**:章节总结、核心发现、行动建议
**Builder 映射**
- 3 点 → `three_col(title, columns=[{head,body},{head,body},{head,body}])`
- 2–4 个 KPI 式要点 → `stats(title, stats=[(value,label,desc),...])`
- 4–7 个要点 → `bullets(title, points=[...])`

**最佳实践**
- 3 点是视觉甜点区
- 每个框的"标题"写结论,"说明"写证据 / 动作
- 这是章节的收束页,一定要明确"所以接下来要干什么"(Bain results orientation)

## 布局选择决策树

```
需要展示一个趋势?              → 1. 标题+单图表型  (PNG+image)
需要对比 2 个方案 / 概念?      → 2. 标题+左右分栏型 (two_col / text_image)
需要战略定位 / 方案取舍?        → 3. 标题+2×2矩阵型 (PNG+image)
需要多维度对比 ≥ 3 列?          → 4. 标题+表格型    (table)
需要拆解增长 / 变化归因?        → 5. 标题+瀑布图型  (PNG+image)
需要展示历史演进 / 路线图?      → 6. 标题+时间轴型  (three_col 或 PNG+image)
需要章节总结 / 核心发现?        → 7. 洞察总结页     (three_col / stats / bullets)
```

## 图表生成约定

当需要"PNG+image"的版式时,Claude 应该:

1. **先写图表生成脚本**(matplotlib 或 plotly),保存到 `/tmp/westwell_charts/<slide_id>.png`
2. **配色用 westwell 色板**:
   ```python
   NAVY = "#1A2B6D"
   TEAL = "#00B3B0"
   ALTROW = "#F3F5F8"
   # 参考 design-system.md
   ```
3. **固定比例**:宽 1920,高按版式需要(单图 1080,半幅 900)
4. **去掉 matplotlib 默认的顶部和右侧边框**(`ax.spines['top'].set_visible(False)`),西井风格偏简洁
5. **字号放大**:轴标签 14pt,刻度 12pt,图例 12pt,保证缩放后仍清晰
6. **生成完成后**,在 slide 里:
   ```python
   ppt.image(title="论点标题", img_path="/tmp/westwell_charts/slide_07.png", caption="数据来源:xxx")
   # 或
   ppt.text_image(title="论点标题", body="洞察句 · 来源:xxx", img_path="/tmp/westwell_charts/slide_07.png")
   ```

**不要让 Claude 每次都重新发明图表脚本的轮子。** 如果一次会话里要生成多张图表,可以先写一个 `charts.py` helper,然后 import 用。

## 与西井视觉风格的平衡

mckinsey 的分析型版式信息密度偏高,而西井整体风格偏大图大留白。**不冲突**,但要注意:

- **每章最多 2–3 张分析型版式**,多了会显得密
- 分析型版式之间用 `statement` / `chapter` / 大图 slide 做视觉缓冲
- 颜色尽量用 westwell 色板,不要用麦肯锡深灰 + 橘色
- 标题仍然 top-left,不要居中(这是西井 ID)
- 表格的表头色用 `C_NAVY` 而不是麦肯锡的纯黑
