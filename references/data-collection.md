# 逐页数据采集 & 生成规范

Phase 3 的核心循环:**一页一页地**做搜索、生成、自检、暂停。
"一次性搜索所有页再一起生成"是反模式,会导致上下文爆炸和证据混串页。

---

## 核心原则

1. **逐页进行**:上一页完全结束后再开始下一页,不要并行
2. **依赖先检查**:生成前先读 Dummy 里该页的"依赖关系"标注(见 `dummy-pages-spec.md`)
3. **证据必须可追溯**:每一条关键数据都要能回到具体 URL 或内部文档名
4. **用完释放**:每一页完成后清掉该页搜索结果,下一页重新开始

---

## 单页生成流程(9 步)

```
对于 Dummy.md 里的每一页:

0. 依赖检查
   - 读取该页的"依赖关系"
   - ✅ 独立 → 继续
   - ⏩ / 📄 / ⏸️ → 先走 `dummy-pages-spec.md` 的依赖检查话术,等用户确认

1. 查看 Dummy 设计
   - 标题(论点式)
   - Slide type
   - 数据需求
   - 信息来源线索

2. 数据收集(2–5 次搜索)
   - 按"信息来源"字段执行
   - 每条记录:关键词 / 找到的数据点 / 完整 URL / 用于哪一页
   - 写入 data-trail.md(见下文)

3. 图表生成(如需)
   - 优先用 `diagram` skill 生成 PNG
   - 不用 diagram 的话,用 matplotlib/plotly 自写脚本
   - 保存到 `/tmp/westwell_charts/<slide_XX>.png`
   - 配色参考 design-system.md

4. 生成本页 PPT 代码
   - 严格按 Dummy 里的 Slide type 和内容
   - 只追加本页的 `ppt.<method>(...)` 调用
   - 不要回头改前面已确认的页面

5. 预览本页
   - 运行 `preview_pptx(..., dpi_scale=1.5)`
   - 把预览 PNG 贴给用户

6. 自检 6 项(见下文)

7. 告知用户:"第 X 页完成,已自检通过。继续下一页吗?"
8. 等待用户确认(可能要求改动 → 迭代本页)
9. 清空该页搜索结果的上下文记忆,进入下一页
```

---

## 自检 6 项(每页必跑)

生成完一页、贴预览给用户之前,Claude 自己先过一遍:

1. **标题是论点,不是 topic label**
   - ❌ "市场现状" → ✅ "港口 AI 调度渗透率突破 40%,中小港口是下一个增量"

2. **Slide type 匹配内容**
   - 数据趋势 → 用单图表型,不要用 bullets 硬写
   - 对比 → 用 two_col / table,不要用 text 堆段落

3. **数据是真实的**(不是编的)
   - 每个数字都能指回 data-trail.md 里的一条记录
   - 如果临时找不到数据,先用占位并注释"[待补:xxx]"

4. **内容量不超量**
   - body 文字 ≤ 60 字 / 每项
   - bullets 条数 ≤ 7
   - 一页只承载一个核心论点

5. **依赖数据已取到**
   - 如果本页依赖前页,确认前页的数据 / 图表已经加载
   - 前页的 chart PNG 路径记下来,复用不重跑

6. **来源已记录**
   - URL 或内部文档名都进了 data-trail.md
   - 图表 caption 带来源(至少精简版本:"来源:xxx")

**自检不通过 → 先改再给用户看。** 不要把已知有问题的页面甩给用户。

## data-trail.md:证据溯源

每个项目一份 `<项目名>_data-trail_<YYYYMMDD>.md`,跟 Dummy 同目录。

### 格式

```markdown
# <项目名> · 数据溯源

## 第 4 页 · 港口 AI 调度渗透率已突破 40%

**论点**: 港口 AI 调度渗透率已突破 40%,头部玩家 ARR 破亿

**数据点 1**: 2024 年渗透率 42%
- 来源: 交通运输部《智慧港口发展白皮书 2024》
- URL: https://example.gov.cn/xxx
- 搜索词: "港口 AI 调度 渗透率 2024"
- 置信度: 高(官方白皮书)

**数据点 2**: 头部玩家 ARR 1.2 亿
- 来源: 36Kr 融资报道
- URL: https://example.36kr.com/xxx
- 搜索词: "港口 AI 调度 融资"
- 置信度: 中(媒体报道,非官方披露)

**图表**: /tmp/westwell_charts/slide_04.png
- 生成脚本: charts.py::plot_penetration_trend()
- 数据来源: 上述两个数据点

---

## 第 10 页 · ...
```

### 为什么保留 data-trail.md

- **答疑备忘**:用户被问"这个数据哪来的",直接查
- **跨对话续写**:新对话里重建证据上下文只需读这份文件
- **自检依据**:生成时每个数据都要有对应条目,强迫 Claude 不编数
- **比 Excel 轻**:纯 Markdown,不需要额外工具,也便于用 `preview-md` 预览

---

## 何时搜索、何时用内部资料

**优先用项目目录下的内部资料**:
- 技术架构、原则、术语 → 项目的技术设计文档
- 项目时间表、Milestone → 项目的立项规划文档
- 场景方案、客户信息 → 项目的场景方案目录

**需要 web_search 的情况**:
- 行业大盘数据、市场规模
- 竞品动态、融资消息
- 最新政策、白皮书引用
- 内部资料里没有的外部验证

**标注惯例**:data-trail.md 里区分来源类型,方便后续审稿:
- `来源(内部)`: 内部 `.md` 文档路径
- `来源(公开)`: URL
- `来源(推断)`: 基于经验的合理假设,必须明确标

## 上下文管理策略

```python
# 每页开始前:当前活跃上下文
current_page = {
    "dummy_design": read_from_dummy_md(page_number),
    "search_results": [],   # 最多 5 条
    "chart_path": None,
    "code_snippet": ""
}

# 每页完成后:写盘 data-trail,清掉内存里的搜索原文
write_data_trail(page_number, current_page)
clear_context(current_page)
move_to_next_page()
```

**关键**:搜索结果原文可能很长,用完就清。data-trail.md 里保留的是**精炼后的条目**(3–5 行 / 条),不是原文复制。

## 断点续写时的 data-trail 处理

新对话里用户上传 Dummy + 已生成的 `.pptx`,建议也让用户带上 data-trail.md:

```
Claude:
"如果你有之前的 data-trail.md,可以一起发我,我就能接上证据链。
没有也没关系,我会把新一轮的证据记在新的 data-trail.md 里。"
```

读取后:
- 已完成的页面直接跳过,不重复搜索
- 前向依赖的数据从 data-trail.md 取,不需要重新找
- 只对要继续生成的页面重新启动数据收集循环
