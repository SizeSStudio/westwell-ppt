# Exemplar · 战略备忘录叙事思路

抽象自 "西井科技战略思考 · 内部咨询版 v1" 一份战略备忘录。**这是思路,不是模板**。
页数、每章内容、每页版式都应根据项目实际需求调整。把这份文件当作"为什么这样讲
有说服力"的原理说明,不是"照这个排就对了"的填空答案。

---

## 触发场景

用户需要一份**说服管理层做艰难战略收敛决定**的材料时,本文思路适用。常见触发:

- "帮我做份给管理层 / 董事会的战略材料"
- "做一份内部咨询备忘录 / 战略思考"
- "IPO 路演前内部讨论用的战略材料"
- 用户给了一份偏散文的战略 `.md`,需要收束为论点结构
- 核心任务是**说服一个权威受众接受一个不舒服的判断**(砍业务、换方向、重定义优先级)

---

## 核心思路:5 个叙事节拍

这类材料的说服力,来自 5 个缺一不可的叙事节拍。顺序、页数、版式都可变,
**节拍必须都在**。

### 节拍 1:核心结论先行(Conclusion First)

**为什么必须先讲结论**:战略材料的受众是高管。他们注意力短、耐心低、
看到 slide 2 就想知道"你到底要我做什么决定"。把结论埋到中后段,受众会
在未被说服前就形成默认立场,后面再讲证据也救不回来。

**这个节拍的表达形式**(任选):
- 一页 `two_col` / `before_after` 做正反对照("我们是什么 vs 我们不能止于此")
- 一页 `pyramid` 把战略分层直接摊开
- 一页 `quote` 把核心主张变成宣言
- 或者两页组合

**判断结论是否到位**:读者看完这一段,能不能用一句话复述你的主张?
不能 → 结论还不够锐利,继续收。

---

### 节拍 2:制造紧迫感(Why Now)

**为什么必须单列这一段**:受众可能已经同意你的结论,但不一定认为"现在就
要改"。**拖延是战略决策的默认选项**,你必须主动击碎它。

**这个节拍要回答**:
- 客户需求 / 市场竞争 / 组织能力正在发生什么迁移?
- 如果继续现状,3 年后我们会在什么位置?(通常是个不舒服的位置)
- 为什么这个窗口期会关闭?

**常用版式**:
- `before_after()` — 客户痛点 / 行业范式迁移
- `two_col(dark=True)` — Path A 继续现状 vs Path B 战略上移
- `big_number()` — 一个关键数据(渗透率、份额、增长率)量化紧迫感
- `stats()` — 2–4 个并列指标

**判断紧迫感是否到位**:受众会不会说"好,我同意这事儿**现在**得谈"?

---

### 节拍 3:主战略分层展开(The Answer)

**为什么必须分层,不能并列罗列**:受众最怕的是听到"我们要做 A、B、C、D、
E"式的战略——因为这等于什么都不收,资源还是会被平均稀释。分层的意义是:
告诉受众**哪个是底盘不能掉、哪个是当前该争、哪个是长期方向不能急**。

**这个节拍要回答**:
- 当前阶段必须守住的是什么?(底盘)
- 中期要争的是什么?(制高点)
- 长期要通向哪里?(北极星)
- 这三层分别占多少资源权重?

**常用版式**:
- `pyramid()` — 一页概览三层战略
- 然后每层用 `bullets` / `number_list` / `three_col` / `two_col` 各展开一页
- 用章节分隔页(`chapter()`)给每一层一个视觉仪式感

**判断分层是否到位**:三层的优先级和资源关系,是不是一目了然?有没有层和
层之间模糊不清的地方?模糊就是危险。

---

### 节拍 4:能力边界与反向清单(Principles & Negatives)

**为什么必须有这一段**:有了战略方向还不够,要落到资源配置原则。这一段
本质是**帮受众做"不做什么"的决定**,比"做什么"的决定更考验魄力。

**这个节拍要回答**:
- 哪些能力是公司必须自控的核心?哪些可以合作?
- 哪些看起来重要、但不能并列成主战略?(咨询能力、内部 AI 化、探索项等)
- 原则是什么?

**常用版式**:
- `control_matrix()` — 必控 vs 可合作
- `not_list()` — 反向清单:什么不是主战略 + 正确定位是什么
- `table()` — 方案多维对比

**判断是否到位**:有没有被直接砍掉 3 个以上的方向?如果读者读完觉得"还是
什么都做",那这一段就没写到位。

---

### 节拍 5:明确的带走项(Bottom Line)

**为什么必须收束**:战略材料不是分析报告。读完后,受众必须能带走**具体的
几条判断**,知道接下来决策时怎么想。

**这个节拍要回答**:
- 现在就要统一的几件事(通常 2–4 条)
- 现在不要误判的几件事(反向列出)
- 一句可以被复述的 bottom line

**常用版式**:
- `two_col()` — 左"要统一的"右"不要误判的"(参考样例)
- `number_list()` — 3–5 条有序建议
- `quote(dark=True)` — 末页收束宣言
- `statement()` — 全幅陈述

**判断是否到位**:受众合上材料后,能不能脱口说出 2–3 条核心判断?如果
不能,这一段太虚了。

---

## 为什么是 5 个节拍,不是更多

这个节拍结构对应 SCQA × 分层战略的融合:

```
Situation     (现状)  ─┐
                       ├→ 节拍 1:核心结论先行(直接跳到 Answer)
Complication  (紧迫)  ─┤
                       ├→ 节拍 2:制造紧迫感(补 S + C)
Question      (提问)  ─┤
                       ├→ 节拍 3:主战略分层(展开 A)
Answer        (答案)  ─┤
                       ├→ 节拍 4:能力边界与反向(A 的原则层)
                       └→ 节拍 5:Bottom line(行动化)
```

少了任何一个节拍,材料都会出问题:
- 少 1 → 受众读完不知道核心主张
- 少 2 → 受众同意但不行动
- 少 3 → 主线散,回到并列罗列
- 少 4 → 有方向没原则,落地时资源还是会稀释
- 少 5 → 受众读完没有带走项

**多了也不好**。超出 5 个节拍的章节,通常是在做调研报告,不是战略备忘录。

## 参考样例:18 页是怎么分配这 5 个节拍的

下面这份 18 页**是一个样例,不是模板**。10 页也能讲完,30 页也能讲完,
关键是每个节拍都到位。

| 节拍 | 样例里用的页数 | 页面作用 |
|------|--------------|---------|
| 1. 核心结论先行 | 3–4 页(封面 + 章节 + 正反对照 + pyramid) | 让受众 30 秒内知道主张 |
| 2. 制造紧迫感 | 3 页(章节 + before_after 痛点迁移 + 规模 vs 上移对比) | 击碎拖延默认 |
| 3. 主战略分层 | 5 页(章节 + 底盘 + 制高点 + 横切维度 + 北极星) | 展开三层 |
| 4. 能力边界 | 3 页(章节 + control_matrix + not_list) | 原则落地 |
| 5. Bottom line | 3 页(章节 + value_ladder + 管理层建议 two_col) | 行动化收束 |

如果你有 10 页要做:1→1 页、2→2 页、3→3 页、4→2 页、5→2 页。
如果你有 30 页要做:每个节拍都可以展开更多证据和中间页,但节拍本身不变。

## 标题写作(论点化)

不管多少页,所有内容页标题必须是**论点**,不是 topic label:

| 弱(topic) | 强(论点) |
|------------|----------|
| 客户需求 | 客户的痛点,已从"缺车"转向"缺运营" |
| 战略分层 | 底盘、制高点、北极星 — 三层战略的分工 |
| 能力边界 | 决定能否进入标准层的能力,必须控在自己手里 |
| 方向排除 | 哪些方向不应作为主战略 |
| 商业模式 | 从卖车到按结果收费的连续升级 |

**标题检查**:只读标题能不能讲出整份材料的故事?不能 → 标题还是 topic
label,改到"读标题就知道论点"为止。

---

## 反向缓冲(Softpedal 技巧)

战略备忘录的独特之处是**每个强论点都要配反向缓冲**。主张太硬 = 显得偏激 = 
受众抵触 = 说服力反而下降。

参考样例里的缓冲手法:

- 主张 "不能只做卖车公司" → 缓冲 "车仍然是基础,必须继续做强"
- 主张 "ReeWell 要争标准层" → 缓冲 "但标准先行不能脱离交付主业"
- 主张 "AI Operator 是终局" → 缓冲 "但不是近两年的主战略"
- 主张 "Ainery 是长期一致性" → 缓冲 "但不能写成独立口号"

**建议**:每 2–3 页插一句"反向缓冲",用 `text()` / chapter subtitle /
quote body / 脚注 italic。这是材料显得**稳、成熟、可信**的关键 — 
它承认了"管理层的顾虑",所以说服力更强。

## 用这份思路做新材料时

1. 先和用户对齐**核心主张一句话**:"我们想让管理层接受 XXX"
2. 再对齐**紧迫感的证据**:"如果不改,3 年后会怎样"
3. 再对齐**分层结构**:"底盘 / 制高点 / 北极星 分别是什么"
4. 以上三条都清楚了,再开始 Dummy 设计
5. Dummy 按 5 个节拍组织,每个节拍分配 1–N 页,总页数跟着内容走
6. 最后两个节拍(能力边界 + bottom line)如果一页能解决就一页解决,
   不要为了凑章节硬拆

**没有先讲清楚 1–3 就动手做 Dummy = 材料会散。** 宁可多花一小时对齐,
少花一天返工。

## 反模式:绝对不要做的事

战略备忘录最常见的稀释方式是"把产品方案 PPT 的套路搬过来"。以下几条是**红线**:

**❌ 不要加 agenda / 目录页**
战略备忘录的读者是高管,不是听会议汇报的下属。他们没耐心看 "01 核心结论 / 02 为什么 / ..." 的目录 — 会觉得你在拖时间。直接上封面 → 章节分隔 → 核心结论。章节分隔页自带"现在到哪一节"的信号,已经足够。

**❌ 不要加 "谢谢 · Q&A" 结尾页**
战略备忘录的收束是 **Bottom line 宣言** — 一句让管理层合上材料能复述的话。"谢谢"是演讲结束语,不是战略材料的归属。用 `quote(dark=True)` 做一页宣言,或者嵌入到最后一页(参见下一条)。

**❌ 不要把 BOTTOM LINE 拆成独立一页**
Claude Design 版的 p18 管理层建议是一个"**建议 + 结论**"复合版面:上面两栏(MUST ALIGN × DO NOT MISJUDGE)+ 底部深蓝 BOTTOM LINE 通栏。这是精心设计的——读者读完 6 条建议立刻看到"所以呢",一页到位,节奏紧。
如果你把 BOTTOM LINE 拆成独立一页,就稀释了这个节奏。

**✅ 正确做法**:用 `two_col(..., bottom_callout={'label': 'BOTTOM LINE', 'text': '车要继续做强,但公司不能止于卖车...', 'dark': True})` 在同一页解决。

**❌ 不要每页标题都加"核心结论"/"主战略" / "能力"这种 topic label**
标题必须是完整论点句。见"标题写作"章节。

**❌ 不要在正文页上堆 6+ bullets**
战略备忘录的每页信息密度 = "一个论点 + 2-4 个支撑点 + 一句 so-what"。超过这个量说明你在写调研报告,不是备忘录。

## Editorial framing:让每页都"像咨询公司出品"

Claude Design 版之所以看起来比"普通 PPT"专业,核心是每页都有**4 层信息编织**:

```
EYEBROW      ← mono caps 小眉标(语义锚点:"核心命题" / "战略分层" / ...)
论点式标题
──────────    ← teal rule
SUBTITLE     ← 14pt 灰色解释段,把标题从"论点"落到"论证"
主视觉 / 卡片 / 列表
FOOTNOTE     ← italic 灰色 so-what 金句收束(可选)
```

pptx_builder 的 `bullets / two_col / three_col / pyramid / value_chain / control_matrix / not_list / before_after / value_ladder / big_number / quote / number_list / step_grid` **全都**接受 `eyebrow / subtitle / footnote / notes` 四个可选参数。

**三条使用规则**:

1. **Eyebrow 必填**(除非页面自成一节)— 这是咨询备忘录的签名动作。Eyebrow 应该是整份材料的"认知索引"(参考 Claude Design 版:核心命题 / 战略分层 / 客户需求迁移 / 核心风险 / 当前底盘 / 中期制高点 / 贯穿主线的一条维度 / 长期北极星 / 能力边界 / 反向清单 / 连续升级路径 / 给管理层的建议)
2. **Subtitle 按需**(约 60% 页面)— 当标题需要"落地"时用。纯事实页/视觉强页可以不用
3. **Footnote 按需**(约 30% 页面)— 当这一页有强烈 so-what 时用。每 2-3 页一个 footnote 是健康节奏

**Bottom_callout 是杀手锏**(only two_col / three_col):
- `FORMULA` 模式:三栏+底部等式(例如 p12 Ainery 页的 "Ainery = AI × 能源维度,嵌入 ReeWell 与调度标准层")
- `BOTTOM LINE` 模式:建议页底部通栏宣言(dark=True 用于 bottom line,dark=False 用于 formula)

**Speaker notes**:用 `notes='...'` 给演讲人留话术提示(不渲染到 slide,但在 PowerPoint 演讲者视图显示)。这是超越 Claude Design 的一个点 — 你的材料除了被读,还能被讲。

## 页面数 benchmark

| 篇幅 | 场景 |
|------|------|
| **≤10 页** | 高管 1-on-1 / 临时说服 / 会议前预读 |
| **15–22 页** | 董事会 / 内部咨询版(本样例属于这一档) |
| **25–35 页** | 正式提报 / 涉及多方利益 / 带外部对标 |
| **>35 页** | 大概率冗长,拆成"执行摘要 + 完整版" |
